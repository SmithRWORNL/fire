\subsection{Detailed Description}
\subsubsection*{template$<$typename Derived$>$class fire\+::\+Tensor\+Provider$<$ Derived $>$}

The \hyperlink{a00047}{Tensor\+Provider} is the base class for injecting 3rd-\/party tensor algebra libraries into the Fire framework. It utilizes the curiously recurring template pattern to enable static polymorphism of \hyperlink{a00047}{Tensor\+Provider} subclasses. The \hyperlink{a00046}{Tensor} class delegates all tensor-\/related data storage and work to the methods provided by \hyperlink{a00047}{Tensor\+Provider}. \hyperlink{a00047}{Tensor\+Provider} takes as a template parameter a derived \hyperlink{a00047}{Tensor\+Provider} and delegates all work to that subclass. It is up to subclasses of \hyperlink{a00047}{Tensor\+Provider} (subclassed with themselves as the \hyperlink{a00047}{Tensor\+Provider} template parameter) to perform the actual work with their representative tensor algebra library.

Subclasses of \hyperlink{a00047}{Tensor\+Provider} must provide a subclass of the \hyperlink{a00032}{Provider\+Builder} struct with the following public method (see \hyperlink{a00062_source}{Eigen\+Tensor\+Provider.\+hpp} for example)\+:


\begin{DoxyCode}
   \textcolor{keyword}{template}<const \textcolor{keywordtype}{int} Rank, \textcolor{keyword}{typename} Scalar>
MyTensorProvider<Rank, Scalar> \hyperlink{a00103_abca66b4f2a1543308b663714bd8b4855}{build}() \{
   MyTensorProvider<Rank, Scalar> prov;
   \textcolor{keywordflow}{return} prov;
\}
\end{DoxyCode}
